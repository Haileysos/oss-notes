
# git  
: 컴퓨터 파일의 변경사항을 추적하고, 여러명의 사용자들 간에 해당 파일들의 작업을 조율할 수 있도록 도와주는 소프트웨어 (version control system : VCS)
<br>  
- 버전관리 : 코드, 문서 등 파일의 변경이력 추적 가능, 각 변경사항에 대해 고유한 커밋 아이디 생성해 기록하고 과거 버전으로 쉽게 되돌리기 및 특정 시점 확인 가능
- 백업 : 분산형 구조로 인해 로컬저장소에 전체 데이터가 저장됨. 중앙서버가 손상되어도 로컬저장소를 통해 복구 가능
- 협업 : 병렬로 작업할 수 있도록 브랜치 생성 및 관리 지원

SSH (secure shell)
☆주로 원격서버에 접속하여 작업을 수행하거나 파일을 전송할 때 사용
컴퓨터 간 안전하게 통신할 수 있도록 하는 네트워크 프로토콜 (원격서버)
통신 내용은 암호화되어 전송되기 때문에 해킹 위험 적음
HTTP/HTTPS
사용자가 웹브라우저에 URL을 입력하면 HTTP/HTTPS 프로토콜을 통해 웹서버와 통신
HTTP는 데이터를 암호화하지 않음
HTTPS는 SSL을 사용해 데이터를 암호화하여 연결 제공

fast-forward merge - 브랜치가 갈라지지 않았으면 새 merge 커밋 없이 main 포인터만 앞으로 이동
rebase - 내 브런치 커밋을 main 최신 커밋 위에 순서대로 다시적용 (replay)
