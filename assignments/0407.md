주절주절 노랑글씨

내가 되돌리려고 하는 파일이 이미 없어  
c1 자체를 취소해버릴려고 하니까 그 파일 자체가 이미 수정? 덮어씌워짐 당해서 없음 찾을 수가 없음  
즉, 내가 되돌리고싶은 파일을 찾을 수가 없어서 노랑글씨로 주절주절 오류나는 거임  


어떻게 되느냐?
- error발생
  - 현재는 resetA.txt 파일의 내용이 수정된 상태(c3)에서 c1 commit을 취소하겠다는 것은  
  - c1에 있었던 resetA.txt 파일을 없애야 하는것이다.
  - 그런데 c1은 수정되기 전의 resetA.txt임
  - 수정되기 전의 resetA.txt는 현재 없음
  - error발생  
ㅡㅡㅡㅡ  
- 충돌 후(error 발생 후) 할 수 있는 옵션들
  - git add 파일이름 : 해당 파일을 남겨둠 -revert한 의미가 없어짐 (c1 commit(resetA.txt추가)을 취소하려고 햇는데 남겨둔다고???)
  - git rm &lt;fileName&gt; : 해당 파일을 없앰  
  - git revert --abort : 해당 revert 모두 중단
  - git revert --skip : 해당 revert만 중단 (여러개의 revert를 하는 경우)

- 해결책  
  - resetA.txt 파일을 없애자!  
  - git rm &lt;fileName&gt;을 한 후에는 git revert --continue로 revert를 마무리 해줘야 함
